This project creates a Microsoft SQL Database which logicaly represents a consultant management system. 


1) The database model consists of the following tables:
Role
Consultant
Client
Project
ProjectConsultantRole
Invoice


2) The database consists of several views listed below to control the way the data is returned to the user. The Views are listed below:
vRole
vClient
vConsultants
vProjects


3) The database contains the following stored procedures in order to encapulsate information in order to make it more modular:
getCOnsultantInfo
getConsultantRolePay
getMinHourlyRateAVG


4)The project contains a number of scripts to populate the table as listed in the file Deploy.cvs.
